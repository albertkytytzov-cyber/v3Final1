# PERFORM Constructor MVP Test Report

Дата: 2026-06-05

Цель: проверить первую сборку ядра конструктора без записи в БД и без автоматического назначения спортсмену.

## Что реализовано

1. `docs/perform-constructor-json-schema.json`
   - формальная JSON-схема входных данных конструктора;
   - соревнование, спортсмен, фаза, цели, тесты, состояние, ограничения.

2. `packages/shared/src/constructor-core.ts`
   - типы конструктора;
   - первые `Cycle Template Cards`;
   - pure-функция `buildPerformConstructorDraft`;
   - функция `buildConstructorTemplatePayload`, которая превращает draft в payload шаблона.

3. `POST /api/v1/plans/constructor/draft`
   - безопасный endpoint;
   - только для coach/admin;
   - проверяет доступ к спортсмену;
   - возвращает draft и templatePayload;
   - ничего не сохраняет и не назначает.

4. `scripts/check-perform-constructor-core.mjs`
   - регрессионная проверка ядра;
   - добавлена в `npm test` как `check:constructor-core`.

## Вывод

Первая сборка ядра работает корректно:

- выбирает подходящие карточки;
- не требует лишних данных, когда тесты есть;
- видит риск веса;
- видит фактор дороги;
- строит templatePayload с днями, сессиями и блоками;
- не назначает план автоматически.
